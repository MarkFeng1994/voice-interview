> 版本: 1.0 | 更新日期: 2026-03-22

## 一、项目概览

**Voice Interview** 是一个基于 AI 的语音模拟面试练习平台，支持用户通过语音或文字与 AI 面试官进行实时对话式面试，系统自动评分并生成面试报告。

### 核心价值

- 真实面试场景模拟：按住说话 + AI 实时追问
- 多维度评估：逐题评分 + 整体报告 + 改进建议
- 简历智能匹配：上传简历自动生成定制面试题
- 多端覆盖：H5 网页 + 微信小程序 + 管理后台

### 技术栈总览

| 层级 | 技术 | 版本 |
|------|------|------|
| 移动端 | uni-app + Vue 3 + TypeScript + Pinia | 3.0 / ^3.4 / ^4.9 / 2.1.7 |
| 管理端 | Vue 3 + Vite + Vue Router + Ant Design Vue | ^3.5 / ^8.0 / ^4.6 / ^4.2 |
| 后端服务 | Spring Boot + Java + Maven | 3.5.7 / 21 / 3.9+ |
| 数据库 | MySQL 8+ | |
| 缓存 | Redis（可选） | |
| 语音服务 | OpenAI / 阿里 DashScope / Sherpa / Mimo | |
| 实时通信 | WebSocket | |
| PDF 解析 | Apache PDFBox | 2.0.31 |

| 调用 | `useInterviewSession` + `useMediaUpload` + `useInterviewSocket` + `useWaveform` |

**面试状态机：**

```
PREVIEW ──[开始面试]──▶ IN_PROGRESS ──[回答/追问循环]──▶ IN_PROGRESS
                              │                              │
                              ├──[跳过最后一题]──▶ COMPLETED   │
                              ├──[手动结束]──────▶ COMPLETED   │
                              └──[所有题完成]────▶ COMPLETED   │
                                                              │
                                                    CANCELLED ◀┘（异常）
```

### 3.3 Composable 详解

#### useInterviewSession — 面试状态机

```typescript
// 状态
messages: Ref<InterviewMessage[]>         // 对话消息列表
sessionStatus: Ref<string>                // PREVIEW | IN_PROGRESS | COMPLETED | CANCELLED
currentQuestionTitle: Ref<string>         // 当前题目标题
currentQuestionPrompt: Ref<string>        // 当前题干
followUpIndex: Ref<number>                // 当前追问轮次
maxFollowUpPerQuestion: Ref<number>       // 每题最大追问数
lastAiAudioUrl: Ref<string>              // 最近 AI 题目音频
textPrompt: Ref<string>                  // 用户输入框内容
progressLabel: Ref<string>               // 进度标签 "1 / 5"

// 方法
startSession(request) → InterviewSessionState  // 启动面试
submitLatestAnswer(fileId?) → session           // 提交回答
refreshSessionState() → void                   // 手动同步
restoreSession(sessionId) → void               // 恢复会话
skipQuestion() → session                       // 跳过当前题
endSession() → session                         // 结束面试
applyIncomingSessionState(session) → void       // 处理 WS 推送
```

#### useInterviewSocket — WebSocket 管理

```typescript
// 状态
socketStatus: Ref<string>   // IDLE | CONNECTING | OPEN | CLOSED | ERROR

// 方法
connect(sessionId) → void   // 建立连接（先获取 ticket）
disconnect() → void         // 断开连接

// WebSocket 消息协议
{ type: 'PING' }           → { type: 'PONG' }
{ type: 'SYNC_REQUEST' }   → { type: 'SESSION_SNAPSHOT', session }
服务端主动推送              → { type: 'SESSION_UPDATED', session }
```

#### useMediaUpload — 录音上传与语音识别

```typescript
// 状态
isRecording: Ref<boolean>              // 录音中
isUploading: Ref<boolean>              // 上传中
lastAudioUrl: Ref<string>             // 最近上传音频 URL
lastUploadedFileId: Ref<string>       // 最近文件 ID
lastTranscript: Ref<string>           // ASR 转写结果
lastAudioDurationMs: Ref<number>      // 音频时长

// 方法
startRecording() → void               // 开始录音（自动选择原生/浏览器）
stopRecording() → void                // 停止录音 → 自动上传
pickAudioFile() → void                // 选择音频文件
transcribeLastAudio() → void          // 重新 ASR 识别
resetMediaState() → void              // 清空媒体状态

// 回调
onAudioSamples(Float32Array) → void   // 实时音频样本（供声波可视化）
onTranscriptReady(string) → void      // 转写完成
```

**浏览器录音流程：**

```
getUserMedia → AudioContext → ScriptProcessorNode
    ↓                              ↓
MediaStream            onaudioprocess 回调
                           ↓              ↓
                   Float32Array[]    onAudioSamples()
                   (缓冲区)          (声波可视化)
                       ↓
              停止 → mergeBuffers → encodeWav → upload
```

#### useWaveform — 声波动画引擎

```typescript
// 状态
amplitudes: Ref<number[]>   // 64 个柱子的振幅值 [0.02 ~ 1.0]
isActive: Ref<boolean>      // 动画是否运行

// 方法
pushSamples(Float32Array)   // 接收音频样本，计算 RMS 振幅
start()                     // 启动 requestAnimationFrame 循环
stop()                      // 停止动画
reset()                     // 重置为静默状态

// 内部行为
- 有声音时：柱子高度随 RMS 振幅实时变化
- 静默时：柱子维持最小高度，呈现缓慢呼吸动画
- 帧率：跟随浏览器 requestAnimationFrame（~60fps）
```

#### useAudioPlayback — 音频播放控制

```typescript
// 方法
playAudio(url: string) → void    // 播放指定 URL 音频
stopAudio() → void               // 停止播放

// 回调
onPlay()                         // 播放开始
onEnded()                        // 播放结束
onError(message)                 // 播放出错
```

### 5.3 面试引擎

#### 核心流程

```
1. 创建会话
   ├── 预设模式：从 InterviewPresetCatalog 加载题目
   └── 简历模式：
       ├── PdfBox 解析 PDF → 纯文本
       ├── AI 提取关键词
       ├── 关键词匹配题库已有题目
       ├── AI 为未匹配关键词生成新题目
       └── 合并为面试计划

2. 面试循环
   ├── AI 发送题目（可选 TTS 音频）
   ├── 用户回答（文字 / 语音上传 → ASR 转写）
   ├── AI 评估回答 + 决定是否追问
   ├── 追问轮次 ≤ maxFollowUpPerQuestion
   └── 进入下一题 / 结束面试

3. 生成报告
   ├── 逐题评分 + AI 分析
   ├── 整体得分
   └── 优势/不足/建议
```

#### 会话存储策略

| 模式 | 配置值 | 适用场景 |

| 内存 | `memory` | 开发/测试，重启丢失 |
| Redis | `redis` | 生产推荐，分布式缓存 |
| JDBC | `jdbc` | 持久化到 MySQL |

## 七、核心业务流程

### 7.1 完整面试流程

```
┌─────────┐    ┌─────────┐    ┌─────────────┐    ┌──────────┐
│  登录    │───▶│  准备    │───▶│  面试进行中   │───▶│  报告    │
│         │    │ setup   │    │  session     │    │ report   │
└─────────┘    └─────────┘    └──────┬──────┘    └──────────┘
                                     │
                              ┌──────▼──────┐
                              │ 循环：       │
                              │ 1.AI 出题    │
                              │ 2.用户回答    │
                              │ 3.AI 追问    │
                              │ 4.下一题     │
                              └─────────────┘
```

### 7.2 语音录制链路

```
用户按住说话
    ↓
getUserMedia() → MediaStream
    ↓
AudioContext + ScriptProcessorNode
    ↓
onaudioprocess 回调
    ├──▶ Float32Array → browserBuffers（累积）
    └──▶ onAudioSamples → useWaveform.pushSamples（声波可视化）
    ↓
用户松开
    ↓
mergeAudioBuffers → encodeWav → File
    ↓
uploadMediaByBrowserFile → POST /api/media/upload
    ↓
ASR 转写 → POST /api/asr/transcriptions/{fileId}
    ↓
转写文本自动填入输入框 → 用户确认发送
```

### 7.3 简历匹配流程

```
上传简历 PDF
    ↓
POST /api/media/upload/resume → fileId
    ↓
POST /api/interviews { resumeFileId, questionCount }
    ↓
后端处理：
  1. PdfBox 解析 PDF → 纯文本
  2. AiService.extractResumeKeywords(text) → 关键词列表
  3. ResumeQuestionMatcher 匹配题库已有题目
  4. AiService.generateResumeQuestions() → 为未匹配项生成新题
  5. 组合面试计划 → 创建 InterviewSession
    ↓
返回 InterviewSessionState → 客户端进入面试
```

### 7.4 WebSocket 实时同步

```
客户端                          服务端
  │                              │
  │──POST /interviews/{id}/ws-ticket──▶│ 签发短期 ticket
  │◀───── { ticket }─────────── │
  │                              │
  │──WS /ws/interview?ticket=xxx─▶│ 验证 ticket → 建立连接
  │◀───── 连接建立 ──────────── │
  │                              │
  │──{ type: "SYNC_REQUEST" }───▶│
  │◀─{ type: "SESSION_SNAPSHOT" }│ 推送完整状态
  │                              │
  │   ... 面试进行中 ...          │
  │                              │
  │                    某用户提交回答 → AI 处理 → 状态变更
  │                              │
  │◀─{ type: "SESSION_UPDATED" }─│ 广播给所有连接的客户端
  │                              │
  │──{ type: "PING" }───────────▶│
  │◀─{ type: "PONG" }───────────│ 心跳保活
```
