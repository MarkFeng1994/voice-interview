# Voice Interview 执行任务清单

## 目标

把 [PLAN.md](E:/developSoft/ideaworkspace/voice-interview/PLAN.md) 拆成可执行任务，优先交付：

- 手机端可用
- 对话式模拟面试
- MVP 为半双工语音问答
- 支持文本兜底
- 支持面试历史与报告查看

---

## 执行原则

- 先打通闭环，再补管理能力
- 先做移动端主链路，再做后台管理端
- 先做半双工，再做全双工
- 每个任务都要有明确产出和验收标准

## 关键路径

必须按顺序推进的主路径：

1. 移动端录音与播放
2. 后端媒体上传与读取
3. ASR 转写
4. LLM 面试编排
5. TTS 返回音频
6. 单题完整问答闭环
7. 多题面试会话 + 报告

可以并行推进的侧路径：

- 用户鉴权
- 题库与分类基础 CRUD
- 历史页与报告页 UI
- 管理端骨架

## M0：方向验证 / 技术 Spike

### `M0-01` 仓库结构初始化

- 任务：建立三端目录结构
- 产出：
  - `voice-interview-mobile/`
  - `voice-interview-backend/`
  - `voice-interview-admin/`
  - 根目录 README / 环境说明
- 依赖：无
- 验收：
  - 三端都能本地启动空工程
  - README 说明启动方式

### `M0-02` 移动端聊天页原型

- 任务：用 `uni-app(Vue 3)` 做手机聊天界面原型
- 产出：
  - 登录后进入首页
  - 面试 `session` 页面静态原型
  - 聊天气泡、录音按钮、音频播放条
- 依赖：`M0-01`
- 验收：
  - Android 真机可打开页面
  - iPhone 真机或模拟器可打开页面
  - 页面结构符合聊天式交互，不是后台表格式页面

### `M0-03` 后端骨架与健康检查

- 任务：初始化 Spring Boot 工程
- 产出：
  - `Spring Boot 3 + Java 17`
  - 基础分层结构
  - `/actuator/health` 或 `/health`
  - 统一响应结构
- 依赖：`M0-01`
- 验收：
  - 服务可本地启动
  - 健康检查正常返回

### `M0-04` 媒体上传与播放链路

- 任务：打通录音文件上传和回放
- 产出：
  - `POST /api/media/upload`
  - `GET /api/media/{fileId}`
  - 本地存储或 MinIO 抽象
  - 移动端录音后可回放
- 依赖：`M0-02`, `M0-03`
- 验收：
  - 真机录音后可上传成功
  - 后端能返回 `fileId`
  - 移动端可以播放刚上传的音频

### `M0-05` ASR 单链路验证

- 任务：接入一个云端 ASR Provider，完成音频转文本
- 产出：
  - `AsrService` 接口
  - 一个实际可用实现
  - demo 接口：传 `fileId` 返回 transcript
- 依赖：`M0-04`
- 验收：
  - 普通中文回答转写成功
  - 转写结果可回显到移动端
  - 失败时返回明确错误码

### `M0-06` LLM + TTS 单链路验证

- 任务：接入一个 LLM 和一个 TTS，完成“文本输入 -> AI 文本 -> AI 音频”
- 产出：
  - `AiService`
  - `TtsService`
  - demo 接口：输入文本返回 AI 文本和 `audioUrl`
- 依赖：`M0-03`
- 验收：
  - AI 能返回结构化面试话术
  - TTS 音频可在手机端播放

### `M0-07` 单题完整问答闭环

- 任务：用硬编码题目跑通一轮完整面试
- 产出：
  - 创建会话
  - AI 首题
  - 用户录音上传
  - ASR
  - LLM 决策
  - TTS 回传
- 依赖：`M0-04`, `M0-05`, `M0-06`
- 验收：
  - 用户能在真机上完成 1 题问答
  - 失败可退回文本回答

### `M0-08` Spike 结论记录

- 任务：沉淀技术验证结果
- 产出：
  - 各链路耗时
  - Android/iOS 差异
  - ASR/TTS 可用性问题
  - 下一阶段参数建议
- 依赖：`M0-07`
- 验收：
  - 文档记录完整
  - 能作为 M1 开发基线

## 当前建议

下一步最合适的动作不是直接铺满全部模块，而是：

1. 先按 `M0` 打通“真机录音 -> ASR -> LLM -> TTS -> 播放”
2. 只要 `M0` 跑不通，`M1` 大量开发都容易返工
3. `M0` 跑通后，再进入 `M1` 的标准工程化开发
